## Key Relationships Summary

### 1. **User Management**
- **User** вҶ’ **Driver** (One-to-One): Each driver is a user with additional driver-specific data
- **User** вҶ’ **Role** (Many-to-One): Users can have roles (student, driver, admin)

### 2. **Bus Management**
- **Bus** вҶ’ **Driver** (Many-to-One): Each bus can be assigned to one driver
- **Bus** вҶ’ **Route** (Many-to-One): Each bus can be assigned to one route
- **Bus** вҶ’ **Location** (One-to-Many): Each bus has multiple location records over time

### 3. **Route Management**
- **Route** вҶ’ **BusStop** (One-to-Many): Each route contains multiple bus stops
- **Route** вҶ’ **Bus** (One-to-Many): Each route can have multiple buses assigned

### 4. **Assignment Management**
- **DriverBusAssignment** (Junction Table): Manages the many-to-many relationship between drivers, buses, and routes

### 5. **Location Tracking**
- **Bus** вҶ’ **LiveLocation** (One-to-Many): Real-time location updates
- **Bus** вҶ’ **LocationHistory** (One-to-Many): Historical location data
- **Location** вҶ’ **ETAInfo** (One-to-One): Estimated arrival information

### 6. **Service Dependencies**
- **WebSocketService** вҶ’ **LocationService**: Real-time location updates
- **AuthService** вҶ’ **DatabaseUser**: User authentication
- **MapService** вҶ’ **BusService**: Map display and bus tracking
- **AdminService** вҶ’ **All Entities**: Administrative operations
