## LEGEND

- **Rectangles** = Entities (Strong Entities)
- **Double Rectangles (в•”в•җв•—)** = Weak Entities (LIVE_LOCATIONS)
- **Ovals** = Attributes
- **Diamonds** = Relationships
- **Double Diamonds (в•”в•җв•—)** = Identifying Relationships
- **Double Lines** = Total Participation
- **Underlined Attributes** = Primary Keys (PK)

---

## ENTITY SUMMARY

### Strong Entities:

1. **USER_PROFILES**
   - Primary Key: `Id`
   - Attributes: Id (PK), Email, Full_Name, Role, Phone, License_No
   - Represents all system users (admins, drivers, students)

2. **BUSES**
   - Primary Key: `Id`
   - Attributes: Id (PK), Bus_Number, Vehicle_No, Capacity, Model, Year
   - Represents physical buses in the fleet

3. **ROUTES**
   - Primary Key: `Id`
   - Attributes: Id (PK), Name, Origin, Destination, Distance_km, Estimated_Duration
   - Represents bus routes with geographic information

### Weak Entity:

4. **LIVE_LOCATIONS** (Double Rectangle)
   - Partial Key: `Id` (depends on parent entities)
   - Attributes: Location (PK), Speed_kmh, Heading_Degrees, Recorded_At
   - Depends on: BUSES and USER_PROFILES (drivers)

### Relationships:

1. **Drives** (Many-to-Many)
   - USER_PROFILES вҶ” BUSES
   - Attribute: `Assigned_At`

2. **Assigns** (Many-to-Many)
   - USER_PROFILES вҶ” ROUTES
   - Attribute: `Assigned_At`

3. **Serves** (Many-to-Many)
   - BUSES вҶ” ROUTES
   - Attribute: `Is_Active`

4. **Tracks** (One-to-Many, Identifying - Double Diamond)
   - BUSES вҶ’ LIVE_LOCATIONS
   - Attribute: `Recorded_At`
   - Total participation on LIVE_LOCATIONS (double line)

5. **Reports** (One-to-Many, Identifying - Double Diamond)
   - USER_PROFILES (Drivers) вҶ’ LIVE_LOCATIONS
   - Attribute: `Recorded_At`
   - Total participation on LIVE_LOCATIONS (double line)

---

## KEY FEATURES

- **LIVE_LOCATIONS** is a weak entity that depends on both BUSES and USER_PROFILES (drivers)
- Identifying relationships (double diamonds) connect LIVE_LOCATIONS to its parent entities
- Total participation (double lines) ensures every location record must be associated with both a bus and a driver
- Many-to-many relationships are managed through the DRIVER_BUS_ASSIGNMENTS concept (not shown as separate entity for simplicity)
- Primary keys are underlined in attribute notation
